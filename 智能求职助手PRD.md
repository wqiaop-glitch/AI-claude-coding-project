## 2. 产品背景与目标

### 2.1 问题陈述

在美国求职的候选人面临以下痛点：

- **信息分散**：需要同时在 LinkedIn、Indeed、Glassdoor 等多个平台浏览岗位，耗时巨大
- **简历版本管理困难**：针对不同方向（如 Marketing、Growth、Operations）准备了多份简历，但不清楚哪份最适合某个特定岗位
- **简历定制效率低**：每个高意向岗位都需要根据 JD 手动调整简历，耗费大量时间
- **投递追踪混乱**：投递了哪些岗位、用了哪份简历、目前状态如何，缺少系统化管理

### 2.2 产品愿景

打造一个**一站式智能求职助手**，每日自动聚合多平台岗位，智能匹配简历，并用 AI 辅助定制简历，让求职者**把精力花在面试准备上，而不是找岗位和改简历上**。

### 2.3 成功指标（KPI）

| 指标 | 目标值 |
|------|--------|
| 用户每日节省求职时间 | ≥ 1 小时 |
| 岗位匹配准确率（用户反馈满意） | ≥ 80% |
| AI 修改简历后用户采纳率 | ≥ 70% |
| 用户月活跃留存率 | ≥ 60% |

## 3. 用户画像（Persona）

**小李 — Marketing 方向求职者**

- 年龄：25-35 岁
- 背景：在美国有市场/运营相关工作经验或留学背景
- 目标岗位：Marketing Manager、Growth Marketing、Digital Marketing、Brand Strategist、Marketing Operations 等
- 持有 2-3 份不同侧重的简历（品牌方向、增长方向、运营方向）
- 痛点：每天花 2+ 小时浏览多个招聘网站，看到好岗位后又要花 30 分钟改简历
- 期望：有工具帮忙聚合岗位、自动推荐简历版本、一键优化简历

## 4. 功能需求

### 4.1 用户账号与简历管理模块

**用户故事**：作为求职者，我希望能注册账号并上传管理多份简历，以便系统根据不同岗位匹配合适的简历版本。

| 功能点 | 描述 | 验收标准 |
|--------|------|----------|
| 注册/登录 | 支持邮箱注册、Google OAuth 登录 | 用户可正常注册、登录、登出 |
| 简历上传 | 支持上传 PDF/DOCX 格式简历，最多 5 份 | 上传成功后可预览简历内容 |
| 简历命名/标签 | 用户可为每份简历添加名称和标签（如"品牌方向"、"增长方向"） | 标签可编辑、删除 |
| 简历解析 | 系统自动提取简历中的技能、经历、教育背景等关键信息 | 解析结果可供用户确认和修正 |
| 简历版本管理 | AI 修改过的简历自动保存为新版本，保留历史记录 | 用户可查看、回滚任一版本 |

### 4.2 职业偏好设置模块

**用户故事**：作为求职者，我希望设定我的求职偏好，以便系统推送最相关的岗位。

| 功能点 | 描述 | 验收标准 |
|--------|------|----------|
| 目标岗位关键词 | 设置目标职位名称关键词（如 Marketing Manager, Growth Lead） | 支持多个关键词，可增删 |
| 目标地点 | 选择工作地点偏好（城市/州/Remote） | 支持多选，含 Remote 选项 |
| 薪资范围 | 设定期望年薪区间 | 可选填，不影响匹配但用于筛选 |
| 行业偏好 | 选择目标行业（Tech、CPG、Healthcare 等） | 多选 |
| 公司规模偏好 | 选择偏好的公司阶段（Startup / Mid / Enterprise） | 多选 |
| 排除条件 | 设置不想看到的公司或关键词 | 包含排除词的岗位不出现在推荐中 |

### 4.3 岗位抓取与聚合模块

**用户故事**：作为求职者，我希望系统每日自动从多个平台收集最新岗位，让我不需要手动浏览多个网站。

| 功能点 | 描述 | 验收标准 |
|--------|------|----------|
| LinkedIn 岗位采集 | 通过 LinkedIn Jobs API 或公开数据获取岗位 | 每日获取符合用户偏好的新岗位 |
| Indeed 岗位采集 | 通过 Indeed API / RSS 获取岗位 | 同上 |
| Glassdoor 岗位采集 | 通过 Glassdoor API 获取岗位 | 同上 |
| Greenhouse/Lever 采集 | 抓取目标公司 ATS 系统的公开岗位页 | 用户可添加目标公司的 ATS 链接 |
| 岗位去重 | 同一岗位在多个平台出现时合并展示 | 去重准确率 ≥ 95% |
| 岗位信息标准化 | 统一字段：职位名称、公司、地点、薪资、JD 全文、发布时间、来源链接 | 所有岗位数据结构一致 |
| 定时执行 | 每日自动执行抓取任务（可配置时间） | 用户每天登录即可看到新岗位 |

### 4.4 岗位匹配与评分模块

**用户故事**：作为求职者，我希望系统告诉我哪些岗位和我最匹配，减少筛选时间。

| 功能点 | 描述 | 验收标准 |
|--------|------|----------|
| 匹配度评分 | 基于用户简历内容和 JD 进行 AI 语义匹配，给出 0-100 分 | 每个岗位显示匹配分数 |
| 匹配维度细分 | 分别展示技能匹配度、经验匹配度、行业匹配度 | 用户可了解匹配/不匹配的具体原因 |
| 智能排序 | 岗位列表默认按匹配度降序排列 | 支持切换为按时间、薪资排序 |
| 匹配说明 | 对每个岗位给出简短的匹配分析文字 | 如："你的 Google Analytics 和 SEO 经验与该岗位高度匹配，但缺少 Salesforce 经验" |

### 4.5 简历推荐模块

**用户故事**：作为求职者，我有多份简历，希望系统告诉我针对某个岗位应该用哪一份。

| 功能点 | 描述 | 验收标准 |
|--------|------|----------|
| 最佳简历推荐 | 针对每个岗位，推荐最适合的简历版本 | 显示推荐简历名称及推荐理由 |
| 简历-岗位对比 | 展示推荐简历与 JD 的关键词匹配对比 | 高亮匹配项和缺失项 |
| 多简历评分 | 用户可查看所有简历对某岗位的匹配得分 | 列表展示，一目了然 |

### 4.6 AI 简历定制优化模块

**用户故事**：作为求职者，当我发现一个高匹配岗位时，希望 AI 帮我针对 JD 优化简历，提高通过率。

| 功能点 | 描述 | 验收标准 |
|--------|------|----------|
| 一键优化 | 用户点击"针对该岗位优化简历"，AI 基于 JD 调整简历内容 | 生成优化后的简历草稿 |
| 优化建议预览 | 展示 AI 的具体修改建议（diff 对比） | 修改部分高亮显示，用户逐条确认 |
| 关键词注入 | 将 JD 中的关键技能词自然融入简历描述中 | 不改变事实，只优化表述 |
| 经历重排序 | 根据 JD 侧重点调整简历中经历的展示顺序和篇幅 | 用户可手动调整 |
| Summary 生成 | 根据 JD 生成定制化的 Professional Summary | 用户可编辑后采用 |
| 在线编辑器 | 提供富文本编辑器，用户可在 AI 建议基础上进一步修改 | 支持撤销/重做，实时预览 |
| 导出 | 优化后的简历可导出为 PDF | 保持专业排版格式 |

### 4.7 投递记录与追踪模块

**用户故事**：作为求职者，我希望记录我的所有投递情况，追踪每个岗位的进展。

| 功能点 | 描述 | 验收标准 |
|--------|------|----------|
| 标记已投递 | 用户可将岗位标记为"已投递"，记录投递日期和使用的简历版本 | 已投递岗位不再出现在推荐列表中 |
| 状态追踪 | 支持状态：待投递 → 已投递 → 面试中 → Offer → 拒绝 | 用户可手动更新状态 |
| 看板视图 | Kanban 风格展示各状态的岗位 | 支持拖拽更改状态 |
| 投递统计 | 展示投递总数、面试率、各平台投递分布 | Dashboard 可视化图表 |
| 备注 | 每个投递记录可添加备注（如面试时间、联系人等） | 支持富文本备注 |

### 4.8 每日推送/通知模块

**用户故事**：作为求职者，我希望每天收到高匹配岗位的通知，不错过任何机会。

| 功能点 | 描述 | 验收标准 |
|--------|------|----------|
| 每日邮件摘要 | 每日发送匹配度 Top 10 岗位的邮件 | 用户可设置接收时间 |
| 站内通知 | 新高匹配岗位（≥80分）时站内提醒 | 登录后可见未读通知 |
| 通知偏好设置 | 用户可设置通知频率和匹配度阈值 | 支持关闭通知 |

## 7. 数据模型

### 核心实体关系

```
User (1) ──── (N) Resume
User (1) ──── (1) JobPreference
User (1) ──── (N) Application

JobPosting (N) ──── (N) MatchScore ──── (N) Resume
JobPosting (1) ──── (N) Application
```

### 字段设计

**User**
```
id, email, name, avatar_url, auth_provider,
created_at, updated_at
```

**Resume**
```
id, user_id, name, label/tags, file_url,
parsed_skills[], parsed_experience[], parsed_education[],
version, parent_resume_id, source_job_id,
created_at, updated_at
```

**JobPreference**
```
id, user_id, target_titles[], target_locations[],
salary_min, salary_max, industries[], company_sizes[],
excluded_companies[], excluded_keywords[],
notification_enabled, notification_time,
created_at, updated_at
```

**JobPosting**
```
id, title, company, location, salary_range,
description_full, requirements[], source_platform,
source_url, posted_at, is_remote,
created_at, updated_at
```

**MatchScore**
```
id, job_id, resume_id, user_id,
overall_score, skill_score, experience_score, industry_score,
match_summary, missing_keywords[],
created_at
```

**Application**
```
id, user_id, job_id, resume_id,
status (待投递/已投递/面试中/Offer/拒绝),
applied_at, notes,
created_at, updated_at
```

## 8. UI/UX 关键页面

### 8.1 Dashboard（首页）

- 今日新增岗位数量统计
- Top 5 高匹配岗位卡片（显示匹配分、推荐简历）
- 投递进度概览（已投递 / 面试中 / Offer 数量）
- 快捷入口：查看全部岗位、管理简历、更新偏好

### 8.2 岗位列表页

- 左侧筛选面板（匹配度区间、平台来源、地点、薪资、发布时间）
- 右侧岗位列表卡片（职位名、公司、地点、匹配分、推荐简历标签、发布时间）
- 排序切换（匹配度 / 最新 / 薪资）
- 批量操作（收藏、忽略、标记已投递）

### 8.3 岗位详情 + 简历匹配页

- 左栏：JD 全文展示，关键词高亮
- 右栏：匹配分析
  - 总分 + 各维度分数
  - 匹配关键词 ✅ 和缺失关键词 ❌
  - 推荐使用的简历及理由
  - "AI 优化简历"按钮
- 底部操作栏：标记已投递 / 收藏 / 忽略

### 8.4 AI 简历编辑器

- 上方：JD 关键信息摘要
- 左栏：原始简历内容
- 右栏：AI 优化后的简历（diff 高亮修改处）
- 用户可逐条接受/拒绝修改
- 底部："全部接受" / "导出 PDF" / "保存为新版本"

### 8.5 投递追踪页

- Kanban 看板视图（列：待投递 / 已投递 / 面试中 / Offer / 拒绝）
- 每张卡片显示：职位名、公司、投递日期、使用的简历
- 可拖拽改变状态
- 统计图表区：投递趋势、平台分布、面试转化率

### 8.6 简历管理页

- 简历卡片列表（名称、标签、上传时间、版本数）
- 点击进入简历详情：查看解析结果、历史版本、关联的投递记录
- 上传新简历入口

## 10. 风险与限制

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 招聘平台反爬/API 限制 | 无法稳定获取岗位数据 | 优先使用官方 API；设置合理抓取频率；多数据源冗余 |
| LinkedIn 封号风险 | 数据来源中断 | 不使用用户账号登录抓取；使用公开 API 和 RSS |
| AI 修改简历引入不实信息 | 损害用户信任 | AI 仅优化表述，明确标注修改处，用户逐条确认 |
| 简历解析准确率 | 匹配评分偏差 | 允许用户人工修正解析结果 |
| 数据隐私合规 | 法律风险 | 遵守 CCPA；数据加密；提供数据删除功能 |
| AI API 成本 | 运营成本高 | 对免费用户限制每日 AI 优化次数；缓存相似岗位的匹配结果 |

| 术语 | 解释 |
|------|------|
| JD | Job Description，岗位描述 |
